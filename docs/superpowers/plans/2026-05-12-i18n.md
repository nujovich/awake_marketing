# i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ES / EN / IT multilingual support to the Astro 6 marketing site using Astro's built-in i18n routing, per-locale page folders with translated slugs, and JSON translation files.

**Architecture:** All user-visible strings move from `content.ts` / `services.ts` into `src/i18n/{locale}.json`. Each locale gets its own page folder (`src/pages/es/`, `en/`, `it/`) with translated URL slugs. Layout receives `locale` and `currentSlug` props, imports all three JSON files, selects the right one, and passes the translations object down to Header and Footer. No component internals change — components only receive translated strings as props.

**Tech Stack:** Astro 6, Tailwind CSS 4, TypeScript. No external i18n library.

## Task 3: Create src/i18n/utils.ts

**Files:**
- Create: `src/i18n/utils.ts`

- [ ] **Step 1: Create the file**

```ts
// src/i18n/utils.ts
import { getRelativeLocaleUrl } from 'astro:i18n';

export const slugMap: Record<string, Record<string, string>> = {
  'sobre-nosotros': { en: 'about-us',  it: 'chi-siamo' },
  'servicios':      { en: 'services',  it: 'servizi'   },
  'contacto':       { en: 'contact',   it: 'contatto'  },
};

const reverseSlugMap: Record<string, Record<string, string>> = {
  'about-us':  { es: 'sobre-nosotros', it: 'chi-siamo'  },
  'services':  { es: 'servicios',      it: 'servizi'    },
  'contact':   { es: 'contacto',       it: 'contatto'   },
  'chi-siamo': { es: 'sobre-nosotros', en: 'about-us'   },
  'servizi':   { es: 'servicios',      en: 'services'   },
  'contatto':  { es: 'contacto',       en: 'contact'    },
};

export function getLocalizedUrl(targetLocale: string, currentSlug: string): string {
  if (!currentSlug) return getRelativeLocaleUrl(targetLocale, '/');
  const mapped =
    slugMap[currentSlug]?.[targetLocale] ??
    reverseSlugMap[currentSlug]?.[targetLocale] ??
    currentSlug;
  return getRelativeLocaleUrl(targetLocale, mapped);
}
```

