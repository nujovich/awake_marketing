# i18n Design — Awake Marketing

**Date:** 2026-05-12
**Status:** Approved
**Languages:** Spanish (es), English (en), Italian (it)

---

## Summary

Add multilingual support to the Astro 6 + Tailwind 4 static marketing site. All three locales get a URL prefix (`/es/`, `/en/`, `/it/`). Spanish is the default: visiting `/` redirects to `/es/`. Page slugs are translated per locale. All user-visible strings move into per-locale JSON files so non-technical editors can maintain translations without touching code.

---

## Architecture

### Astro i18n config (`astro.config.mjs`)

```js
i18n: {
  defaultLocale: 'es',
  locales: ['es', 'en', 'it'],
  routing: { prefixDefaultLocale: true }
}
```

Astro handles:
- The `/` → `/es/` redirect automatically.
- Setting `<html lang>` based on `Astro.currentLocale`.
- The `getRelativeLocaleUrl()` helper for building locale-aware URLs.

### File structure

```
src/
  i18n/
    es.json          ← all Spanish strings
    en.json          ← English translations
    it.json          ← Italian translations
    utils.ts         ← slug map + getLocalizedUrl() helper
  types.ts           ← Tier type (moved from services.ts)
  pages/
    index.astro      ← redirects / → /es/ via Astro i18n
    es/
      index.astro
      sobre-nosotros.astro
      servicios.astro
      contacto.astro
    en/
      index.astro
      about-us.astro
      services.astro
      contact.astro
    it/
      index.astro
      chi-siamo.astro
      servizi.astro
      contatto.astro
  data/
    content.ts       ← REMOVED (text migrated to JSON)
    services.ts      ← REMOVED (text migrated to JSON; Tier type → types.ts)
```

## Component Pattern

Page files are the only locale-aware layer. Components are locale-agnostic and receive translated strings as props — no changes to component internals.

```astro
---
// src/pages/en/about-us.astro
import Layout from "../../layouts/Layout.astro";
import t from "../../i18n/en.json";
import ValuesSection from "../../components/ValuesSection.astro";
---
<Layout title={t.brand.full} locale="en" currentSlug="about-us">
  <ValuesSection values={t.about.valores} mision={t.about.mision} />
</Layout>
```

`Layout.astro` receives two new props:
- `locale: string` — used for `<html lang>` (or derived from `Astro.currentLocale`)
- `currentSlug: string` — passed to `Header.astro` for the language switcher

## Language Switcher

Added to `Header.astro`. Renders as `ES · EN · IT` — three text links with a dot divider. Active locale: `text-paper`. Inactive: `text-paper/40 hover:text-paper`. No flags, no dropdown.

On mobile, the switcher appears at the bottom of the existing mobile nav drawer.

### Slug resolution (`src/i18n/utils.ts`)

```ts
import { getRelativeLocaleUrl } from 'astro:i18n';

export const slugMap: Record<string, Record<string, string>> = {
  'sobre-nosotros': { en: 'about-us',  it: 'chi-siamo' },
  'servicios':      { en: 'services',  it: 'servizi'   },
  'contacto':       { en: 'contact',   it: 'contatto'  },
};

export function getLocalizedUrl(targetLocale: string, currentSlug: string): string {
  if (!currentSlug) return getRelativeLocaleUrl(targetLocale, '/');
  const mapped = slugMap[currentSlug]?.[targetLocale] ?? currentSlug;
  return getRelativeLocaleUrl(targetLocale, mapped);
}
```

From `/en/about-us`, clicking `IT` resolves to `/it/chi-siamo`. Clicking `ES` resolves to `/es/sobre-nosotros`.

## Content Migration

`content.ts` and `services.ts` are deleted. Every user-visible string migrates to the JSON files. The `Tier` type moves to `src/types.ts`.

**Zero translatable strings remain hardcoded** in `.astro` files or TypeScript modules after migration. This includes:
- All inline strings in `index.astro`, `sobre-nosotros.astro`, `servicios.astro`, `contacto.astro`
- Marquee words, yes/no lists, section labels
- CTA button labels, nav items
- All tier names, taglines, feature lists, badge labels

## SEO — hreflang meta tags

`<link rel="alternate" hreflang>` tags are generated in `Layout.astro` for every page, covering all three locales plus an `x-default` pointing to Spanish.

### Configuration

The canonical domain is set once in `astro.config.mjs` via Astro's built-in `site` option:

```js
export default defineConfig({
  site: 'https://awakemkt.com', // update this when the production domain is confirmed
  i18n: { ... }
})
```

`Layout.astro` reads it via `Astro.site` — no other file needs to change when the domain changes.

### Output (example: English "about" page)

```html
<link rel="alternate" hreflang="es" href="https://awakemkt.com/es/sobre-nosotros" />
<link rel="alternate" hreflang="en" href="https://awakemkt.com/en/about-us" />
<link rel="alternate" hreflang="it" href="https://awakemkt.com/it/chi-siamo" />
<link rel="alternate" hreflang="x-default" href="https://awakemkt.com/es/" />
```

`x-default` points to Spanish as the global default. All URLs are absolute (required by Google).

### Implementation in `Layout.astro`

```astro
---
const locales = ['es', 'en', 'it'];
---
{locales.map(loc => (
  <link rel="alternate" hreflang={loc}
    href={`${Astro.site}${getLocalizedUrl(loc, currentSlug)}`} />
))}
<link rel="alternate" hreflang="x-default" href={`${Astro.site}es/`} />
```
